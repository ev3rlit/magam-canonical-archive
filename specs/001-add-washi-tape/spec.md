# 기능 명세서: 와시 테이프 오브젝트

**기능 브랜치**: `001-add-washi-tape`  
**작성일**: 2026-03-01  
**상태**: 초안  
**입력**: 사용자 설명: "`docs/features/washi-tape/README.md`"

## 사용자 시나리오 및 테스트 *(필수)*

### 사용자 스토리 1 - 빠른 강조 라벨 생성 (우선순위: P1)

캔버스 사용자는 여러 오브젝트를 수동으로 조합하지 않고 한 번에 테이프 형태 강조 요소를 만들 수 있다.

**우선순위 근거**: 이 기능의 핵심 가치이며, 현재 가장 빈번한 수작업 흐름을 직접 대체한다.

**독립 테스트**: 프리셋 스타일과 텍스트를 포함한 와시 테이프 오브젝트 1개를 추가하고, 단일 선택 가능한 오브젝트로 보이는지 확인하면 독립적으로 검증 가능하다.

**인수 시나리오**:

1. **Given** 사용자가 캔버스를 편집 중이고, **When** 프리셋 스타일과 라벨 텍스트를 가진 와시 테이프를 삽입하면, **Then** 입력한 텍스트가 포함된 단일 강조 오버레이가 표시된다.
2. **Given** 사용자가 와시 테이프가 포함된 문서를 저장 후 다시 열면, **When** 문서가 재로딩될 때, **Then** 각 테이프의 스타일, 배치, 콘텐츠가 유지된다.
3. **Given** 사용자가 캔버스 toolbar의 와시 프리셋 버튼을 열면, **When** 프리셋 목록을 확인할 때, **Then** 시스템이 사전 제공하는 `PresetPattern` 카탈로그가 식별자와 함께 노출되어 즉시 참조 가능해야 한다.

---

### 사용자 스토리 2 - 브랜드 맞춤 테이프 스타일 적용 (우선순위: P2)

디자이너는 단색, 벡터 패턴, 이미지 텍스처 기반의 커스텀 스타일을 적용해 브랜드 또는 프로젝트 맥락에 맞는 테이프 비주얼을 구성한다.

**우선순위 근거**: 프리셋만으로 표현할 수 없는 시각 시스템을 가진 팀에 필수적인 요구사항이다.

**독립 테스트**: 서로 다른 커스텀 패턴 유형 3가지를 적용한 테이프를 생성하고, 기존 캔버스 상호작용을 깨뜨리지 않으면서 모두 렌더링되는지 확인하면 된다.

**인수 시나리오**:

1. **Given** 사용자가 유효한 커스텀 스타일 소스를 제공하고, **When** 이를 와시 테이프에 적용하면, **Then** 캔버스와 재열기 이후 모두 해당 스타일로 렌더링된다.
2. **Given** 사용자가 미지원 또는 잘못된 스타일 소스를 제공하고, **When** 테이프를 렌더링하면, **Then** 시스템은 안전한 기본 스타일로 대체 표시하며 세션은 중단 없이 계속된다.

---

### 사용자 스토리 3 - 정밀 배치 및 내보내기 일관성 확보 (우선순위: P3)

사용자는 끝점 기반, 길이/방향 기반, 다른 오브젝트 상대 부착 방식으로 와시 테이프를 배치하고, 내보내기에서도 동일한 시각 결과를 기대한다.

**우선순위 근거**: 실무 문서 제작 흐름에서는 배치 제어와 내보내기 일관성이 필수다.

**독립 테스트**: 모든 배치 방식으로 테이프를 생성하고, 부착 대상 오브젝트를 이동한 뒤, 지원 포맷 전체로 내보내기해 검증할 수 있다.

**인수 시나리오**:

1. **Given** 테이프가 다른 오브젝트에 상대 부착되어 있고, **When** 대상 오브젝트가 이동 또는 크기 변경되면, **Then** 테이프는 상대 부착 상태를 유지한다.
2. **Given** 프리셋과 커스텀 테이프가 혼합된 캔버스에서, **When** 사용자가 PNG, JPEG, SVG, PDF로 내보내면, **Then** 내보낸 결과가 캔버스 뷰와 시각적으로 일관되어야 한다.

### 엣지 케이스

- 배치 입력이 0 또는 0에 가까운 길이로 계산되면 어떻게 처리되는가?
- 부착 대상 오브젝트가 삭제되거나 숨김 처리되면 시스템은 어떻게 동작하는가?
- 라벨 콘텐츠가 테이프 영역을 초과하면 어떻게 처리되는가?
- 하나의 장면에 와시 테이프가 100개 이상일 때 렌더링/조작은 어떻게 동작하는가?
- 문서 재열기 시 외부 에셋을 불러올 수 없으면 어떻게 처리되는가?
- 사전 제공 `PresetPattern` 일부가 로드 실패하거나 누락된 경우 삽입/렌더링은 어떻게 보장되는가?

## 요구사항 *(필수)*

### 기능 요구사항

- **FR-001**: 시스템은 기존 스티커와 독립적으로 삽입, 선택, 관리할 수 있는 전용 와시 테이프 오브젝트 타입을 제공해야 한다.
- **FR-002**: 시스템은 사전 정의된 스타일 프리셋으로 사용자가 와시 테이프를 빠르게 생성할 수 있어야 한다.
- **FR-003**: 시스템은 단색, 벡터 패턴, 이미지 기반 옵션을 하나의 일관된 구성 모델로 커스텀 스타일 입력할 수 있어야 한다.
- **FR-004**: 시스템은 문서 저장 및 재열기 후에도 와시 테이프의 배치, 스타일, 콘텐츠를 보존해야 한다.
- **FR-005**: 시스템은 끝점-끝점 배치, 원점/길이/방향 배치, 상대 부착 배치의 3가지 배치 모드를 지원해야 한다.
- **FR-006**: 시스템은 참조 대상 오브젝트의 위치, 크기, 방향이 변경되어도 상대 부착 배치를 유지해야 한다.
- **FR-007**: 시스템은 와시 테이프에 대해 키보드 기반 삽입, 선택, 포커스 이동을 지원해야 한다.
- **FR-008**: 시스템은 v1에서 전용 인스펙터 패널 없이 기존 퀵 액션 및 명령 팔레트 흐름으로 스타일 변경을 지원해야 한다.
- **FR-009**: 시스템은 v1에서 삽입, 선택, 다중 선택 상호작용을 지원하되, 직접 변형/편집 핸들은 제공하지 않아야 한다.
- **FR-010**: 시스템은 스타일 입력이 잘못되거나 불완전해도 사용자 세션을 중단하지 않고 안전한 대체 렌더링을 제공해야 한다.
- **FR-011**: 시스템은 같은 문서 상태에서 자동 계산되는 방향이 항상 결정적으로 유지되어 재로딩 시 예기치 않은 각도 변화를 만들지 않아야 한다.
- **FR-012**: 시스템은 와시 테이프 내부 텍스트/콘텐츠 오버레이를 지원하고 가독성을 유지하는 일관된 오버플로 정책을 적용해야 한다.
- **FR-013**: 시스템은 캔버스 렌더링과 PNG, JPEG, SVG, PDF 내보내기 결과 간 시각적 일관성을 유지해야 한다.
- **FR-014**: 시스템은 동일 문서 내에서 스티커와 와시 테이프가 함께 존재해도 파싱 또는 렌더링 회귀 없이 동작해야 한다.
- **FR-015**: 시스템은 v1에서 사전 제공 `PresetPattern` 카탈로그를 내장하고 캔버스 toolbar 버튼을 통해 목록을 확인할 수 있어야 하며, 각 프리셋은 안정적인 식별자 기반으로 선택/저장/재열기 시 동일하게 복원되어야 한다.

### 핵심 엔티티 *(데이터가 관련된 경우 포함)*

- **와시 테이프 항목**: 식별 정보, 레이어 순서, 배치, 스타일, 사용자 콘텐츠를 담는 강조 오버레이 오브젝트.
- **패턴 스타일**: 프리셋, 단색, 벡터 패턴, 이미지 소스 중 하나로 정의되며 테이프 표면 표현을 결정하는 스타일 정의.
- **PresetPattern 카탈로그**: v1에서 제품이 기본 제공하는 프리셋 패턴 집합. 각 항목은 고유 ID, 시각 토큰 정의, fallback 우선순위를 가진다.
- **배치 규칙**: 절대 배치 또는 상대 부착 동작을 표현하는 정규화된 위치 규칙.
- **부착 대상 참조**: 와시 테이프가 상대 위치 업데이트를 위해 다른 캔버스 오브젝트를 참조하는 연결 정보.
- **내보내기 표현**: 지원 출력 포맷에서 와시 테이프 항목이 보존/표현되는 결과물.

## 성공 기준 *(필수)*

### 측정 가능한 결과

- **SC-001**: 사용성 테스트에서 사용자 90% 이상이 추가 보조 오브젝트 없이 45초 이내에 프리셋 기반 와시 테이프 라벨을 생성한다.
- **SC-002**: 강조 라벨 생성 시간의 중앙값이 기존 수동 다중 오브젝트 방식 대비 최소 40% 단축된다.
- **SC-003**: 혼합 오브젝트 장면 200개 회귀 테스트에서 와시 테이프의 배치, 스타일, 콘텐츠 보존율이 100%를 달성한다.
- **SC-004**: 내보내기 인수 테스트에서 PNG, JPEG, SVG, PDF 전 포맷 기준 샘플의 98% 이상이 캔버스와 동등한 시각 결과로 판정된다.
- **SC-005**: 키보드 전용 작업 테스트에서 사용자 95% 이상이 마우스 없이 삽입-선택-스타일 변경 흐름을 완료한다.
- **SC-006**: 릴리스 후 첫 4주 동안 와시 테이프 관련 치명적 크래시는 0건을 유지한다.

## 가정

- 기존 스티커 동작은 변경하지 않으며 병행 지원한다.
- v1 범위에서는 와시 테이프 전용 인스펙터 패널과 캔버스 직접 변형 편집을 제외한다.
- 커스텀 스타일 입력을 안전하게 렌더링할 수 없을 경우 기본 프리셋 스타일이 항상 제공된다.
- 브랜드 맞춤 커스텀 스타일 에셋은 각 팀이 제공하고 관리한다.

## 의존성

- 기존 캔버스 삽입, 선택, 키보드 포커스, 명령 팔레트 흐름을 재사용할 수 있어야 한다.
- 기존 문서 저장/재열기 흐름이 새 오브젝트 타입 추가를 포맷 마이그레이션 장애 없이 수용해야 한다.
- 기존 내보내기 파이프라인이 새 오브젝트 타입을 PNG, JPEG, SVG, PDF로 렌더링할 수 있어야 한다.

## 범위 제외

- v1에서 와시 테이프 전용 인스펙터 패널 제공.
- v1에서 와시 테이프의 캔버스 직접 변형 편집 핸들 제공.
- 커스텀 패턴 라이브러리 업로드/큐레이션을 위한 에셋 관리 워크플로.
