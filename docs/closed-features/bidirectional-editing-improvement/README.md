# Bidirectional Editing Improvement

## 배경

현재 양방향 편집 기능은 한 번에 너무 많은 문제를 해결하려고 하면서 실제 사용성과 안정성이 떨어지고 있습니다.

이번 개선은 다음 전제를 유지합니다.

- Magam의 정체성은 "코드로 캔버스를 그린다"이다.
- TSX 코드가 Source of Truth이다.
- 양방향 편집은 "이미 만들어진 오브젝트를 코드로 다시 편집하기 쉬운 범위"에 집중한다.

## 목표

- 편집 액션을 최소한의 코드 변경으로 반영한다.
- 사용자가 예측 가능한 방식으로 코드를 재편집할 수 있게 한다.
- 큰 구조 변경보다, 안전하고 실효성 높은 편집 경험을 먼저 완성한다.

## v1 범위 (Focused Editing)

1. 절대 좌표 노드 이동 시 `x`, `y` 숫자 값만 변경
2. Markdown/일반 텍스트를 `children`으로 가진 컴포넌트의 텍스트 편집 (Markdown은 WYSIWYG)
3. attach된 Sticker/Washi Tape의 상대 위치 변경

## 상세 요구사항

### 1) 절대 좌표 노드 위치 편집

- 캔버스에서 절대 좌표 기반 노드를 이동하면 코드의 `x`, `y` 값만 갱신한다.
- 다른 prop(`id`, `style`, `children`, `from` 등)은 변경하지 않는다.
- 이동 반영은 드래그 종료 시점 기준으로 수행한다.

### 2) Markdown/텍스트 children 편집

- 텍스트 편집 대상은 `children` 문자열로 표현되는 Markdown/일반 텍스트 컴포넌트다.
- 편집 결과는 해당 컴포넌트의 `children` 내용만 갱신한다.
- Markdown 편집은 WYSIWYG 방식으로 제공해 편집 중 시각 결과를 즉시 확인할 수 있어야 한다.
- 텍스트/Markdown 컴포넌트가 여러 개일 때는 현재 선택한 컴포넌트만 편집되어야 한다.
- 비선택 컴포넌트의 `children`은 현재 편집 작업으로 변경되면 안 된다.
- 텍스트 편집 시 컴포넌트 구조나 다른 prop은 변경하지 않는다.

### 3) Sticker/Washi Tape 상대 위치 편집

- attach된 Sticker/Washi Tape의 상대적 오프셋(부모 기준 위치)만 변경한다.
- 부모-자식 attach 관계는 유지한다.
- 상대 위치 변경 시 절대 좌표 재작성 대신 attach 표현 모델을 우선 유지한다.

## 비범위 (이번 단계에서 하지 않음)

- 신규 오브젝트 생성/삭제
- 대규모 구조 재배치(reparent 포함)
- 여러 속성 동시 편집(스타일, 타입 변경 등)
- 자동 레이아웃/정렬 규칙 변경

## 수용 기준 (Acceptance Criteria)

- 노드 이동 후 코드 diff가 `x`, `y` 수정으로만 나타난다.
- Markdown/텍스트 편집 후 코드 diff가 해당 `children` 텍스트 변경으로만 나타난다.
- Markdown 편집 중 표시된 WYSIWYG 결과와 저장 후 렌더 결과가 일치해야 한다.
- 텍스트/Markdown 컴포넌트가 여러 개인 경우, 선택한 컴포넌트 외에는 편집 영향이 없어야 한다.
- Sticker/Washi Tape 이동 후 코드 diff가 상대 위치 값 변경으로만 나타난다.
- 위 3가지 편집에서 의도하지 않은 prop/구조 변경이 없어야 한다.
