# Magam - Milestone Plan

## 문서 목적

이 문서는 `docs/milestones/README.md`와 `docs/milestones/USECASE.md`를 바탕으로,
제품 컨텍스트를 실제 개발 순서와 완료 기준으로 변환한 실행용 마일스톤 문서이다.

이 문서의 목적은 다음과 같다.

- 무엇을 어떤 순서로 만들지 정한다.
- 각 단계에서 반드시 제공해야 할 사용자 가치를 고정한다.
- 마일스톤마다 포함 범위와 제외 범위를 명확히 한다.
- 외부 AI 협업 기능보다 편집 안전성을 우선하는 기준을 명시한다.

---

## 제품 방향 요약

Magam은 로컬 파일 시스템 기반 데스크톱 캔버스 앱이다.
핵심은 앱 내부에 AI를 넣는 것이 아니라, 사용자가 이미 보유한 외부 AI 도구가
캔버스를 안전하게 편집할 수 있도록 만드는 것이다.

따라서 이 제품의 로드맵은 다음 순서를 따른다.

1. 로컬 기반 작업 환경을 안정적으로 만든다.
2. 사람이 직접 쓰기 좋은 캔버스 편집기를 완성한다.
3. 재사용 가능한 템플릿/컴포넌트 체계를 만든다.
4. 외부 편집 주체가 접근할 수 있는 연결과 권한 구조를 만든다.
5. 외부 변경을 안전하게 검토, 승인, 복구할 수 있게 만든다.
6. 충돌 해결, 히스토리, 확장성까지 포함한 협업형 편집 환경으로 확장한다.

---

## 마일스톤 설계 원칙

- `Mobile First`: 모든 핵심 흐름은 좁은 화면에서도 수행 가능해야 한다.
- `Local First`: 로컬 폴더, 파일 접근, 저장 신뢰성이 가장 먼저 확보되어야 한다.
- `Human First Editing`: 외부 협업보다 먼저 사람이 직접 편집 가능한 기본기가 완성되어야 한다.
- `Safety Before Automation`: 연결 자체보다 권한, 승인, 로그, 롤백이 우선이다.
- `Actor Model`: AI는 앱 내부 기능이 아니라 외부 편집 주체로 다룬다.
- `Vertical Slice`: 각 마일스톤은 데모 가능한 사용자 흐름 단위로 마감한다.

## 마일스톤 개요

### Milestone 1. 로컬 워크스페이스 기반

**목표**

사용자가 로컬 폴더를 워크스페이스로 등록하고, 앱을 안전하게 시작할 수 있는 기반을 만든다.

**핵심 사용자 가치**

- 앱을 처음 실행하고 바로 작업 공간을 등록할 수 있다.
- 파일 접근 권한과 로컬 폴더 상태를 신뢰할 수 있다.
- 최근 작업과 워크스페이스 진입이 자연스럽다.

**포함 범위**

- 앱 홈 / 초기 화면
- 온보딩 / 첫 실행 가이드
- 권한 / 로컬 폴더 접근 설정
- 워크스페이스 대시보드
- 워크스페이스 추가 / 삭제 / 검색 / 정렬
- 최근 항목의 최소 기능

**제외 범위**

- 외부 도구 연결
- 변경 검토 / 승인
- 버전 히스토리
- 템플릿 / 컴포넌트 고도화

**완료 기준**

- 사용자가 로컬 폴더를 워크스페이스로 등록할 수 있다.
- 권한 거부, 경로 변경, 접근 실패 상황을 인지 가능한 방식으로 처리한다.
- 워크스페이스 목록을 그리드 / 리스트로 볼 수 있다.
- 최근 접근 항목으로 재진입할 수 있다.

---

### Milestone 2. 코어 캔버스 편집기

**목표**

외부 협업이 없어도 독립적으로 가치가 있는 캔버스 편집기를 완성한다.

**핵심 사용자 가치**

- 사용자가 직접 캔버스를 만들고 편집할 수 있다.
- 모바일 우선 구조에서도 핵심 편집 흐름이 유지된다.
- 저장 상태를 신뢰할 수 있다.

**포함 범위**

- 워크스페이스 상세 / 캔버스 목록
- 새 캔버스 생성 / 삭제 / 복제 / 이동 / 보관의 기본 흐름
- 캔버스 편집기
- 캔버스 뷰포트
- 툴바 / 하단 액션바
- 속성 패널 / 레이어 패널
- 저장 상태 표시

**제외 범위**

- 외부 편집 상태 레이어
- 변경 승인 배너
- 충돌 해결 UI
- 활동 로그 / 스냅샷

**완료 기준**

- 사용자가 워크스페이스 안에서 캔버스를 생성하고 편집할 수 있다.
- 기본 오브젝트 선택, 배치, 수정 흐름이 동작한다.
- 모바일 우선 레이아웃 전환 구조가 깨지지 않는다.
- 저장 상태와 실패 상태가 분명하게 표시된다.

---

### Milestone 3. 재사용 자산 체계

**목표**

캔버스를 반복 생산할 수 있도록 템플릿과 컴포넌트 라이브러리의 최소 체계를 만든다.

**핵심 사용자 가치**

- 기존 작업물을 재사용 가능한 시작점으로 바꿀 수 있다.
- 반복적으로 쓰는 오브젝트를 라이브러리처럼 다룰 수 있다.

**포함 범위**

- 템플릿 라이브러리
- 기존 캔버스를 템플릿으로 저장
- 템플릿으로 새 캔버스 생성
- 컴포넌트 라이브러리
- 검색 / 필터 / 미리보기의 최소 기능
- 삽입 가능한 재사용 오브젝트 체계

**제외 범위**

- 플러그인 마켓플레이스 수준의 확장 관리
- 외부 도구가 템플릿/컴포넌트를 직접 수정하는 고급 흐름

**완료 기준**

- 사용자가 템플릿을 저장, 삭제, 재사용할 수 있다.
- 사용자가 컴포넌트를 탐색하고 캔버스에 삽입할 수 있다.
- 워크스페이스 생성 흐름과 템플릿/컴포넌트 흐름이 자연스럽게 연결된다.

---

### Milestone 4. 외부 편집 주체 연결

**목표**

외부 AI 도구를 앱 내부 기능이 아니라 연결된 편집 주체로 다루는 협업 기반을 만든다.

**핵심 사용자 가치**

- 사용자는 어떤 외부 도구가 어느 범위까지 편집 가능한지 통제할 수 있다.
- 외부 도구 연결이 제품의 중심 기능이 아니라 권한 기반 협업 설정으로 보인다.

**포함 범위**

- 연결 / 협업 설정 화면
- 연결 가능한 외부 도구 목록 구조
- 워크스페이스별 허용 범위 설정
- 읽기 전용 / 편집 가능 / 삭제 허용 등 권한 정책
- 승인 정책의 기본값
- 세션 상태 표시
- 테스트 연결

**제외 범위**

- 세밀한 diff 기반 검토 화면
- 히스토리 기반 복구
- 고급 충돌 병합

**완료 기준**

- 외부 도구를 연결된 편집 주체로 등록/관리할 수 있다.
- 워크스페이스 단위 또는 전역 단위 편집 권한을 설정할 수 있다.
- 연결 상태와 세션 상태를 사용자에게 노출할 수 있다.
- UI 용어가 AI 기능이 아니라 연결 / 협업 / 외부 편집 주체 중심으로 유지된다.

---

### Milestone 5. 승인 기반 외부 편집

**목표**

외부 도구가 만든 변경을 사용자가 신뢰할 수 있도록 검토와 승인 흐름을 만든다.

**핵심 사용자 가치**

- 외부 변경이 자동으로 덮어쓰지 않는다.
- 사용자가 무엇이 어떻게 바뀌는지 이해한 뒤 반영 여부를 결정할 수 있다.

**포함 범위**

- 캔버스 편집기 내 외부 편집 상태 레이어
- 변경 제안 배너
- 변경 검토 / 승인 화면
- 변경 리스트
- 오브젝트 단위 diff
- 개별 승인 / 거부 / 전체 적용
- 변경 출처 표시

**제외 범위**

- 완전한 버전 관리 시스템
- 장기 보관용 고급 감사 리포트

**완료 기준**

- 외부 변경이 제안 형태로 사용자에게 표시된다.
- 사용자는 변경 단위를 검토하고 승인 또는 거부할 수 있다.
- 삭제, 대규모 이동, 다수 변경 같은 고위험 작업은 승인 정책과 연결된다.
- 편집기에서 현재 협업 상태를 즉시 파악할 수 있다.

---

### Milestone 6. 활동 기록, 히스토리, 복구

**목표**

외부 협업 편집을 지속적으로 사용할 수 있도록 로그, 스냅샷, 롤백 구조를 완성한다.

**핵심 사용자 가치**

- 누가 무엇을 언제 바꿨는지 추적할 수 있다.
- 잘못된 변경을 되돌릴 수 있다.
- 사람 편집과 외부 편집이 섞여도 안전하다.

**포함 범위**

- 활동 기록 / 편집 로그
- 사용자 / 외부 도구 / 자동 동기화 기준 필터
- 편집 상세 보기
- 되돌리기 / 재적용
- 버전 히스토리 / 스냅샷
- 비교 보기 / 복원 / 이름 붙여 저장

**제외 범위**

- 멀티유저 SaaS 협업
- 원격 서버 동기화

**완료 기준**

- 사용자와 외부 도구의 변경 이력을 타임라인 형태로 볼 수 있다.
- 의미 있는 시점으로 복원할 수 있다.
- 승인된 변경과 직접 편집이 같은 히스토리 체계 안에서 추적된다.

---

### Milestone 7. 충돌 해결과 확장 단계

**목표**

실사용 단계에서 발생하는 충돌, 오류, 확장성 문제를 다룰 수 있는 안정화 구간을 만든다.

**핵심 사용자 가치**

- 로컬 파일 기반에서 발생하는 현실적인 충돌을 해결할 수 있다.
- 확장 기능을 추가하더라도 기본 편집 모델이 흔들리지 않는다.

**포함 범위**

- 오류 / 파일 충돌 해결 화면
- 병합 / 덮어쓰기 / 분기 저장
- 실패 원인 표시와 재시도 흐름
- 플러그인 / 확장 관리
- 휴지통, 검색 / 점프 패널 같은 확장 화면

**제외 범위**

- 별도 SaaS 협업 플랫폼화
- 앱 내장 AI 챗 경험

**완료 기준**

- 외부 변경과 로컬 변경의 충돌을 감지하고 해결 흐름으로 유도할 수 있다.
- 파일 접근 문제와 손상 가능성을 사용자에게 명확히 설명할 수 있다.
- 확장 기능이 연결 / 권한 / 승인 모델을 우회하지 않는다.

---

## 권장 구현 순서

실제 개발은 아래 순서를 기본으로 한다.

1. `Milestone 1`: 로컬 워크스페이스와 권한 기반을 먼저 완성한다.
2. `Milestone 2`: 사람이 직접 쓰는 편집기의 핵심 가치를 만든다.
3. `Milestone 3`: 템플릿과 컴포넌트로 생산성을 올린다.
4. `Milestone 4`: 외부 편집 주체 연결과 권한 모델을 도입한다.
5. `Milestone 5`: 승인 기반 외부 변경 흐름으로 신뢰를 만든다.
6. `Milestone 6`: 로그, 히스토리, 복구로 편집 안전성을 강화한다.
7. `Milestone 7`: 충돌 해결과 확장 관리로 실사용 안정성을 높인다.

이 순서를 따르는 이유는,
이 제품의 차별점이 외부 AI 연결 자체가 아니라
외부 편집을 포함해도 무너지지 않는 편집 안전성에 있기 때문이다.

## 마일스톤별 핵심 화면 매핑

- `Milestone 1`: 홈, 온보딩, 권한 설정, 워크스페이스 대시보드, 최근 항목
- `Milestone 2`: 워크스페이스 상세, 캔버스 목록, 캔버스 편집기
- `Milestone 3`: 템플릿 라이브러리, 컴포넌트 라이브러리
- `Milestone 4`: 연결 / 협업 설정
- `Milestone 5`: 편집기 협업 상태 레이어, 변경 검토 / 승인
- `Milestone 6`: 활동 기록 / 편집 로그, 버전 히스토리 / 스냅샷
- `Milestone 7`: 오류 / 충돌 해결, 플러그인 / 확장 관리, 휴지통, 검색 / 점프 패널

## 지금 기준의 제품 정의

이 제품의 초기 성공 기준은 다음과 같다.

- 사용자가 로컬 워크스페이스를 안정적으로 등록하고 관리할 수 있다.
- 사용자가 직접 캔버스를 편집하는 경험이 먼저 완성되어 있다.
- 외부 도구는 앱 내부 AI 기능이 아니라 연결된 편집 주체로 보인다.
- 외부 변경은 권한, 승인, 로그, 복구 체계 안에서만 반영된다.
- 사람 편집과 외부 편집이 함께 존재해도 신뢰 가능한 캔버스 환경이 유지된다.

이 문서를 기준으로 이후에는 각 마일스톤을 다시 쪼개
페이지 명세, 상태 정의, 컴포넌트 구조, 구현 계획으로 전개한다.
